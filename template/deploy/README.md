# 项目部署规范

> 这个文件是项目自己的部署文档入口，编写时应遵守通用部署 SOP。

## 1. 适用范围

说明这个部署文档适用于哪个环境、哪些服务、哪些角色。

## 2. 前置条件

至少说明：

- 服务器要求
- Docker 与 Compose 版本要求
- 部署用户与目录要求
- 镜像仓库访问要求

## 3. 目录与文件约定

至少说明：

- 项目部署目录
- 环境文件位置
- 数据目录或命名卷策略
- 日志查看方式

## 4. 环境变量说明

至少说明：

- 真实生产变量来源
- `deploy/env/app.prod.env` 中哪些变量必须替换
- 哪些变量属于敏感信息

## 5. 部署前检查

标准入口：

```bash
make deploy-check
```

## 6. 标准发布流程

标准入口：

```bash
make deploy
```

需要明确写出项目是否还需要：

- 数据库迁移
- 静态资源发布
- 缓存预热
- 网关刷新

## 7. 日志与健康检查

建议至少写清：

- 查看哪个服务日志
- 使用什么命令
- 健康检查接口是什么
- 发布完成后如何确认服务可用

## 8. 回滚流程

标准入口：

```bash
make rollback
```

必须补充：

- 回滚单位
- 回滚命令
- 回滚后验证方式

## 9. 常见故障排查

至少覆盖：

- 环境文件缺失
- 端口冲突
- 容器启动失败
- 健康检查失败
- 数据库未就绪

## 10. 项目特有说明

把这个项目自己的特殊部署前置条件、依赖或限制写在这里。
